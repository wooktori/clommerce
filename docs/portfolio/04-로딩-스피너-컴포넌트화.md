# 로딩 스피너 컴포넌트화 — 흩어진 로딩 UI 통일

## Situation

로딩 상태를 표현하는 방식이 화면마다 제각각이었다.

- `(buyer)`/`(seller)` 레이아웃의 인증 로딩 게이트: `로딩 중...` 텍스트
- 상품 목록의 무한 스크롤 하단 로더: `Loading` 텍스트
- 저장/삭제/결제/로그인/회원가입/주문 취소·상태 변경 버튼: `저장 중...`, `처리 중...` 같은 텍스트 전환

같은 "로딩 중"이라는 의미를 8~9곳에서 서로 다른 마크업으로 반복 구현하고 있어
일관성이 없고, 새로운 로딩 지점이 추가될 때마다 패턴이 또 갈라질 여지가 있었다.

## Task

- 반복되는 텍스트 기반 로딩 표시를 하나의 `Spinner` 컴포넌트로 통일
- 단, 목록/그리드형 로딩은 이미 `.skeleton` 플레이스홀더(디자인 시스템에 정의됨)를 쓰고 있어
  콘텐츠 레이아웃을 미리 보여주는 skeleton과 순간적인 동작을 나타내는 spinner의 UX 목적이 다르므로
  skeleton은 스코프에서 제외
- 밝은 배경(페이지 게이트, 무한 스크롤 하단)과 어두운 배경(검정/빨강 버튼) 양쪽에서
  자연스럽게 보이도록 색상 오버라이드 가능해야 함

## Action

### 1. Spinner 컴포넌트 설계 (`src/components/ui/Spinner.tsx`)

상태·이벤트가 없는 순수 프레젠테이션 컴포넌트라 `"use client"` 없이 서버 컴포넌트로 작성했다.
`size`로 굵기·크기를, `className`으로 색상을 오버라이드할 수 있게 해 다양한 배경에 대응했다.

```tsx
interface SpinnerProps {
  size?: "sm" | "md";
  className?: string;
}

export default function Spinner({ size = "md", className = "" }: SpinnerProps) {
  const dimension = size === "sm" ? "w-4 h-4 border-2" : "w-8 h-8 border-[3px]";

  return (
    <div
      role="status"
      aria-label="로딩 중"
      className={`inline-block rounded-full animate-spin border-rule border-t-ink ${dimension} ${className}`}
    />
  );
}
```

### 2. 적용 범위 확정 — skeleton과의 경계 정리

목록/그리드 skeleton(`SellerProductList`, `RelatedProducts`, `ShopPage` 그리드 등)은 그대로 두고,
텍스트로만 로딩을 표시하던 지점만 교체 대상으로 좁혔다.

- 전체 페이지 로딩 게이트 2곳: `(buyer)/layout.tsx`, `(seller)/layout.tsx`
- 무한 스크롤 하단 로더 2곳: `ShopPage.tsx`, `SellerProductList.tsx`
- 버튼 내 로딩 상태 8곳: 상품 저장/삭제, 결제, 로그인, 회원가입, 주문 취소, 주문 상태 변경

### 3. 다크 배경 버튼 대응 — className 오버라이드

기본 스피너는 밝은 배경(`border-rule border-t-ink`) 기준으로 만들었기 때문에
검정/빨강 배경 버튼 위에서는 흰색 variant로 덮어썼다.

```tsx
{loading ? (
  <Spinner size="sm" className="border-white/30 border-t-white" />
) : (
  submitLabel
)}
```

버튼은 텍스트 → 스피너로 내용이 바뀌어도 크기가 흔들리지 않도록
`inline-flex items-center justify-center`를 함께 추가했다.

### 4. row 단위로 스코프된 mutation과의 호환 확인

`BuyerOrderList`/`SellerOrderList`는 리스트 항목(`OrderRow`)마다 각자
`useCancelOrder()`/`useUpdateOrderStatus()` 훅을 호출하는 구조라
`isPending`이 클릭한 행에만 스코프된다는 걸 먼저 확인한 뒤 적용했다.
(전역 mutation이었다면 한 곳만 클릭해도 모든 행의 스피너가 동시에 돌았을 것이다.)

## Result

- 8~9곳에 흩어져 있던 텍스트 기반 로딩 표시를 `Spinner` 컴포넌트 하나로 통일
- skeleton(콘텐츠 플레이스홀더)과 spinner(순간 동작 표시)의 역할을 분리해 UX 의도를 명확히 함
- `size`/`className` 두 개의 최소 prop만으로 밝은 배경·어두운 배경 양쪽 재사용 가능
- 작업 과정에서 발견한 기존 참조 에러(`ProductForm.tsx`의 `PRODUCT_CATEGORIES` 오타,
  `SellerOrderList.tsx`의 미정의 `STATUS_LABEL`)도 함께 수정해 관련 화면의 런타임 크래시 제거
