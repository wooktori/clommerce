# 결제 플로우 구현 — PortOne V2 SDK 연동

## Situation

장바구니 기능은 구현되어 있었지만 실제 결제로 이어지는 플로우가 없었다.
"구매하기" 버튼을 눌러도 다음 단계가 존재하지 않아 커머스 플랫폼의 핵심 기능이 미완성 상태였다.

추가로 발생할 수 있는 문제들이 다음과 같이 존재했다.

- 재고가 1개인 상품을 두 명이 동시에 구매하면 둘 다 성공할 수 있는 **race condition**
- 결제 실패/취소 시 이미 감소된 재고가 복구되지 않아 **재고 유실** 발생 가능

## Task

- PortOne(아임포트) V2 SDK를 연동해 실제 결제창을 호출
- 결제 전 재고를 원자적으로 선점하고, 결제 실패 시 복구하는 롤백 로직 구현
- 결제 성공 후 주문 문서를 Firestore에 생성하고 장바구니를 초기화

## Action

### 1. PortOne V2 SDK 연동 (`src/lib/iamport.ts`)

`@portone/browser-sdk` npm 패키지를 사용해 스크립트 태그 없이 모듈로 임포트.
V1 방식(`window.IMP`)에서 V2 방식(`PortOne.requestPayment`)으로 전환해
Promise 기반의 깔끔한 비동기 처리가 가능해졌다.

```ts
const response = await PortOne.requestPayment({
  storeId,
  channelKey,
  paymentId: `payment-${Date.now()}`,
  orderName,
  totalAmount,
  currency: "CURRENCY_KRW",
  payMethod: "CARD",
  customer: { fullName, phoneNumber, email },
});

if (response?.code) throw new Error(response.message);
```

### 2. 재고 원자적 선점 — Firestore `runTransaction`

결제 전에 재고를 먼저 감소시켜 다른 구매자가 같은 재고를 선점하지 못하도록 했다.
`runTransaction`으로 read → validate → write를 하나의 원자적 작업으로 처리해
동시 요청에서도 재고 초과 판매가 발생하지 않는다.

```ts
export async function decreaseStock(productId: string, qty: number) {
  await runTransaction(db, async (tx) => {
    const ref = doc(db, "products", productId);
    const snap = await tx.get(ref);
    const current = snap.data()?.productQuantity ?? 0;
    if (current < qty) throw new Error("재고가 부족합니다.");
    tx.update(ref, { productQuantity: current - qty });
  });
}
```

### 3. 결제 플로우 설계 (`CheckoutClient.tsx`)

결제 실패/취소 시 이미 선점한 재고를 복구하는 롤백 패턴을 적용했다.

```
1. 상품별 최신 재고 확인 (getProduct)
2. 모든 상품 재고 순차 감소 (decreaseStock)
   └─ 실패 시: 이미 감소된 항목 전체 복구 후 중단
3. PortOne 결제 요청
   └─ 실패/취소 시: 전체 재고 복구 (restoreStock)
4. 성공 시: 상품별 주문 문서 생성 → 장바구니 초기화 → /buyer/orders 이동
```

### 4. 배송 정보 폼 유효성 검사

`react-hook-form` + `zod`로 수령인·연락처·배송지를 검증.
연락처는 정규식(`/^[0-9]{10,11}$/`)으로 숫자만 허용.

## Result

- 장바구니 → 배송 정보 입력 → 결제창 → 주문 완료까지 전체 E2E 플로우 완성
- Firestore `runTransaction`으로 동시 구매 시 재고 초과 판매 원천 차단
- 결제 실패/취소 시 재고 자동 복구로 데이터 정합성 유지
- 결제 성공 후 Firestore `orders` 컬렉션에 주문 문서 자동 생성
