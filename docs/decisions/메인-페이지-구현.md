# 메인 페이지 구현

## Situation

커머스 플랫폼의 진입점으로서 카테고리별 상품을 노출하고, 구매자가 원하는 카테고리로
바로 이동할 수 있는 랜딩 페이지가 필요했다.

## Task

- 카테고리별 상품 4개씩 노출 (상의, 하의, 아우터, 신발, 악세서리)
- 더보기 버튼으로 카테고리 전체 목록 페이지 이동
- 이미지 여러 장 상품은 캐러셀로 표시
- SSR로 구현해 초기 HTML에 상품 데이터 포함 (SEO + 초기 로드 속도)

## Action

### SSR 적용 방식

Next.js App Router에서 `page.tsx`는 기본적으로 서버 컴포넌트다.
`async/await`로 Firestore를 서버에서 직접 호출하면 별도 설정 없이 SSR이 적용된다.

```tsx
// src/app/page.tsx
export default async function Home() {
  const results = await Promise.allSettled(
    CATEGORIES.map((cat) => getProductsByCategory(cat, 4)),
  );
  // ...
}
```

`Promise.all` 대신 `Promise.allSettled`를 선택한 이유:
`Promise.all`은 하나라도 실패하면 전체가 reject되어 메인 페이지 전체가 500 오류를 반환한다.
`Promise.allSettled`는 실패한 카테고리만 빈 배열로 처리해 나머지 카테고리는 정상 노출된다.

### 컴포넌트 구조

```
page.tsx (RSC, SSR)
└── CategorySection (RSC) — 카테고리 헤더 + 더보기 링크 + 그리드
    └── ProductCard (RSC) — 상품 정보 (이름, 가격, 카테고리)
        └── ProductImageCarousel (Client Component) — 이미지 슬라이드
```

서버/클라이언트 경계를 트리 말단에 배치해 RSC 영역을 최대화했다.
이벤트 핸들러(이전/다음 버튼)가 필요한 `ProductImageCarousel`만 `"use client"`로 분리했다.

### 캐러셀 구현 — Embla Carousel 선택

직접 구현(`useState` + 인덱스 관리)에서 Embla Carousel 라이브러리로 전환했다.

| 항목          | 직접 구현                   | Embla Carousel                       |
| ------------- | --------------------------- | ------------------------------------ |
| 슬라이드 전환 | 즉시 교체 (애니메이션 없음) | CSS transform 기반 부드러운 슬라이드 |
| 터치/스와이프 | 미지원                      | 기본 지원                            |
| 번들 크기     | 0KB                         | ~3KB                                 |

Swiper(~30KB)보다 가볍고 이미지 캐러셀 용도에 충분해 Embla를 선택했다.

### 코드 리뷰 후 수정 사항

코드 리뷰를 통해 발견된 문제를 수정했다.

**1. `<a>` 안에 `<button>` 중첩 (HTML 명세 위반)**

초기 구현에서 `ProductCard` 전체를 `<Link>`로 감싸고 내부에 캐러셀 버튼을 두었다.
이는 `<a>` 안에 interactive element를 중첩할 수 없다는 HTML 명세를 위반하고,
버튼 클릭 시 페이지 이동이 함께 트리거되는 버그를 일으켰다.

```tsx
// 수정 전 (버그)
<Link href={`/products/${id}`}>       ← <a>
  <ProductImageCarousel />             ← 내부에 <button> 포함
</Link>

// 수정 후
<div>
  <ProductImageCarousel productId={id} />   ← 이미지 클릭은 내부 Link로 처리
  <Link href={`/products/${id}`}>           ← 텍스트 정보만 Link
    상품명, 가격...
  </Link>
</div>
```

**2. `Promise.all` → `Promise.allSettled`**

카테고리 하나의 Firestore 쿼리가 실패해도 나머지 카테고리는 정상 노출되도록 변경.

**3. 캐러셀 인디케이터 활성 상태 반영**

초기 구현에서 하단 점 인디케이터가 모두 동일한 색상이었다.
`emblaApi.on("select")` 이벤트로 현재 슬라이드 인덱스를 추적해 활성 점을 구분했다.

**4. 접근성(a11y)**

이전/다음 버튼에 `aria-label` 추가, 인디케이터 점을 클릭 가능한 `<button>`으로 변경.

## Result

- 메인 페이지 SSR 적용: 서버에서 상품 데이터가 포함된 HTML을 반환해 초기 로딩 속도 개선
- 카테고리별 4개 상품 노출 + 더보기 버튼 → `/products?category=상의`
- Embla Carousel로 부드러운 이미지 슬라이드 + 터치 스와이프 지원
- `Promise.allSettled`로 일부 카테고리 오류 시에도 페이지 전체 장애 방지

### 참고: Firestore 복합 인덱스 필요

`getProductsByCategory`는 `productCategory(where) + createdAt(orderBy)` 조합을 사용하므로
Firestore 복합 인덱스 등록이 필요하다. 쿼리 실행 시 콘솔에 인덱스 생성 링크가 출력된다.
