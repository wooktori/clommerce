# Firestore 복합 인덱스

## Situation

판매자 상품 목록 조회 시 데이터가 반환되지 않고 `data`가 계속 `undefined` 상태였다.
네트워크 탭에서 Firestore 요청이 1분 이상 응답 없이 대기 중인 것을 확인했다.

쿼리 코드는 아래와 같다.

```ts
where("sellerId", "==", sellerId)
orderBy("createdAt", "desc")
limit(10)
```

## Task

- 왜 쿼리가 응답을 반환하지 않는지 원인을 파악해야 했다
- Firestore 콘솔에 별도 에러 메시지가 표시되지 않아 원인 파악이 어려웠다

## Action

**Firestore 인덱스 동작 방식을 파악했다.**

Firestore는 데이터를 빠르게 찾기 위해 인덱스(미리 정렬된 목록)를 사용한다.

- **단일 필드 인덱스**: 문서 저장 시 각 필드에 대해 자동 생성
- **복합 인덱스**: 두 개 이상의 필드를 동시에 사용하는 쿼리에 필요, 자동 생성되지 않음

`where`(필터)와 `orderBy`(정렬)를 **서로 다른 필드에 동시에 사용**하면 복합 인덱스가 필요하다.
인덱스가 없으면 Firestore는 쿼리를 실행하지 못하고 응답 자체를 반환하지 않는다.
(느린 게 아니라 응답이 오지 않는 것)

Firebase Console → Firestore → 색인 탭에서 아래 복합 인덱스를 직접 생성해 해결했다.

| 컬렉션 | 필드 | 순서 |
|---|---|---|
| `products` | `sellerId` | 오름차순 |
| `products` | `createdAt` | 내림차순 |

## Result

**복합 인덱스 생성 후 쿼리가 정상적으로 응답을 반환했다.**

Firestore에서 필터(`where`)와 정렬(`orderBy`)을 다른 필드에 함께 사용하는 쿼리는
반드시 복합 인덱스를 직접 생성해야 한다.
인덱스 누락 시 에러 메시지 없이 응답이 오지 않는 경우가 있으므로
쿼리가 무한 대기 상태에 빠지면 복합 인덱스 누락을 먼저 의심해야 한다.
